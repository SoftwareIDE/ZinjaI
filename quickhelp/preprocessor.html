Directivas de Preprocesador

No son instrucciones de un programa propiamente dichas, sino que son directivas interpretadas por el compilador durante el parseo que sirven para indicar que, como o cunado debe compilar, o para realizar verificaciones. Todas comienzan con el caracter numeral (#) y no llevan punto y coma (;) al final de la misma.

Directivas mas comunes:
  • #include: Incluye el código de otro archivo.
  • #define: Define una constante o macro de preprocesador.
  • #undef: Anula la definicion de una constante o macro de preprocesador.
  • #if: Incluye un segmento de código si se cumple una condición.
  • #ifdef: Incluye un segmento de codigo si esta definida una macro o constante de preprocesador.
  • #ifndef: Incluye un segmento de codigo si no esta definida una macro o constante de preprocesador.
  • #elif: Incluye un bloque de código condicional alternativo sino se cumple la condición planteada anteriormente por #if, #ifdef o #ifndef.
  • #else: Incluye un bloque de código alternativo sino se cumple la condición planteada anteriormente por #if, #ifdef o #ifndef.
  • #endif: Cierra un bloque de codigo condicional.
  • #error: Genera un error en tiempo de compilación.
  • #warning: Genera una advertencia en tiempo de compilación.
  • #line: Permite controlar el nombre de archivo y numero de linea con que se informará un error de compilación.
  • #pragma: indica opciones específicas de un determinado compilador.
